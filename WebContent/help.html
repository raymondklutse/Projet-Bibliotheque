Avant toute chose...


Objectifs de la page d'aide

Ce projet est une application Web et comme toute application de ce type, un nombre important de fichiers sont présents. Le paragraphe suivant présente succintement l'organisation du projet, vous verrez rapidement qu'au sein de celui-ci, votre travail est limité à un nombre réduit de fichiers.

Les différentes rubriques de cette page d'aide sont là pour vous aider à faire le lien entre les différentes fonctionnalités de l'application (les différentes pages accessibles par le menu) et le code Java sur lequel vous devez intervenir : quelle(s) méthode(s) de quelle(s) classe(s) est(sont) nécessaire(s) pour que la fonctionnalité soit remplie. Quand le code à compléter est identifié, vous trouverez dans les commentaires Javadoc des méthodes les informations nécessaires sur le format des données à renvoyer.


Organisation de l'application et quelques conseils...

Cette application comporte deux parties :

  1. L'interface utilisateur : vous n'avez pas à priori à y toucher à moins que vous ne sachiez ce que vous faites !

    Cette interface utilisateur est codée au sein de fichiers qui sont tous dans le répertoire WebContent. Elle est constituée principalement de pages JSP (Java Server Pages) qui utilisent une technologie Java permettant de composer dynamiquement des pages Web avec le langage Java. Cette technologie est similaire à la technologie PHP.

    L'affichage Web de cette interface graphique utilise un framwork maintenant très répandu, Bootstrap, voir : http://getbootstrap.com. Ce framework facilite grandement la mise en page de sites Web et est utilisable avec l'intégralité des langages utilisés pour faire du Web dynamique : Java, PHP, Python, etc.
    Comme tout site Web utilisant le framework Bootstrap, l'application bibliothèque est dite responsive, son contenu s'adapte automatiquement au type de terminal utilisé : ordinateur, tablette, téléphone, etc.

    De nombreux thèmes Bootstrap sont disponibles (libres ou payants). Le thème utilisé pour le projet bibliothèque est le thème SB Admin 2, voir : https://startbootstrap.com/template-overviews/sb-admin-2/.
    Le thème SB Admin 2 et les fichiers nécessaires au framework Bootstrap sont localisés dans le répertoire WebContent/bootstrap. Une démonstration des différents composants graphiques fournis par le thème SB Admin 2 est consultable en suivant le lien Démo du template SB Admin 2 dans le menu de l'application.

    Cette interface utilisateur dialogue avec le coeur de l'application qui assure les fonctionnalités métier.

  2. La partie métier de l'application :
    C'est le travail principal que vous aurez à faire dans ce projet. Vous aurez à compléter un certain nombre de classes présentes dans le répertoire src.

    Le dialogue entre la partie métier et l'interface de l'application se réalise par l'appel de méthodes dans les fichiers JSP : voir par exemple le JSP liste_livres.jsp à la ligne 7 qui appelle la méthode listeTousLesLivres() de la classe ComposantBDLivre.

    Ce dialogue est figé, en conséquence : VOUS NE DEVEZ PAS MODIFIER LA SIGNATURE DES MÉTHODES, IL VOUS EST INTERDIT DE :

    • MODIFIER LE NOM DES MÉTHODES
    • MODIFIER LES PARAMÈTRES DE CELLES-CI
    • MODIFIER LE TYPE DE RETOUR

    Des commentaires Javadoc sont présents pour chacune des méthodes sur lesquelles vous aurez à travailler. Ces commentaires vous indiquent précisément les valeurs rendues qu'il est impératif de respecter pour un fonctionnement normal de l'application.

    Afin de vous aider, un page d'erreur spécifique a été développée. Elle vous permettra d'accéder facilement à la ligne de code où l'erreur s'est déclenchée. Attention, il est possible que cette page d'erreur ne soit pas efficace en permanence : il vous faudra alors vous aider des fonctionnalités d'Eclipse afin de localiser l'erreur puis de la résoudre.

    Pour finir, le lien Démonstration du prototype dans le menu vous permet de tester le comportement final de l'application tel qu'il devrait se présenter à la fin de votre projet.

Page d'accueil de l'application

JSP : WebContent/accueil.jsp

La page d'accueil affiche des informations quantitatives sur le contenu de la bibliothèque. On y trouve trois composants graphiques qui affichent le nombre de livres, d'abonnés et d'emprunt en cours. Par défaut, la valeur affichée est -1 tant que les méthodes correspondantes n'ont pas été complétées.
Un lien est présent dans le composant, il permet, comme dans le menu, d'accéder aux listes correspondantes.

Les méthodes à compléter pour réaliser les fonctionalités sont les suivantes :

Classe Méthode
Affichage du nombre de livres ComposantBDLivre nbLivres()
Affichage du nombre d'abonnés ComposantBDAbonne nbAbonnes()
Affichage du nombre d'emprunts en cours ComposantBDEmprunt nbEmpruntsEnCours()
Attention, une méthode similaire existe mais est employée dans un autre endroit de l'application : nbEmpruntsEnCours(int idAbonne).

La démonstration de l'application montre deux courbes affichant jour par jour le nombre d'emprunts et le nombre de retours. Ce travail est optionnel, s'il vous intéresse, vous trouverez plus d'informations sur la page d'aide Statistiques des emprunts.

Liste des livres

JSP : WebContent/liste_livres.jsp

La fonctionnalité d'affichage des livres est celle qui vous demandera le moins de travail, elle est déjà réalisée : la méthode listeTousLesLivres() de la classe ComposantBDLivre vous est donnée à titre d'exemple.

La liste des livres est présentée dans un tableau permettant de faire un certain nombre de manipulations sur l'ensemble des données : recherche, tris, etc.
Pour chacun des livres de la liste, un lien (colonne Informations) permet d'accéder à une page autorisant la manipulation de celui-ci : modification, ajout/suppression d'exemplaire, etc.

Informations sur un livre

JSP : WebContent/infos_livre.jsp

La page donnant les informations sur un livre offre plusieurs fonctionnalités : gestion du livre à gauche de la page et gestion des exemplaires associés à droite.

Gestion du livre

Sur la partie gauche, un formulaire s'affiche. Il reprend les informations du livre (isbn10 et 13, titre, etc.), elles peuvent ensuite être modifiées. Le livre peut aussi être supprimé à l'aide de ce formulaire.

Les méthodes nécessaires pour remplir ces fonctionnalités sont :

Classe Méthode
Récupération des informations sur le livre ComposantBDLivre getLivre(int idLivre)
Modification des informations ComposantBDLivre modifierLivre(int idLivre, String isbn10, String isbn13, String titre, String auteur)
Suppression du livre ComposantBDLivre supprimerLivre(int idLivre)

Gestion des exemplaires

La gestion des exemplaires en partie droite affiche le nombre total d'exemplaires associés au livre et pour chacun d'entre-eux, leur disponibilité (emprunté ou non). Deux fonctionnalités sont aussi disponibles : il est possible d'ajouter un nouvel exemplaire et de supprimer un exemplaire donné.

Les méthodes nécessaires sont :

Classe Méthode
Afficher le nombre d'exemplaires associés au livre ComposantBDLivre nbExemplaires(int idLivre)
Afficher la liste des exemplaires associés au livre ComposantBDLivre listeExemplaires(int idLivre)
Ajouter un nouvel exemplaire pour le livre ComposantBDLivre ajouterExemplaire(int idLivre)
Suppression d'un exemplaire donné ComposantBDLivre supprimerExemplaire(int idExemplaire)
Afficher la disponibilité d'un exemplaire donné ComposantBDEmprunt estEmprunte(int idExemplaire)

Insertion d'un livre

JSP : WebContent/inserer_livre.jsp

La page d'insertion d'un livre affiche un simple formulaire permettant la saisie des données nécessaires à l'identification d'un livre. Une fois les données saisies et le livre enregistré dans la base de données, ces informations sont affichées sur la page.

Les méthodes nécessaires sont :

Classe Méthode
Insertion d'un nouveau livre dans la base de données ComposantBDLivre insererNouveauLivre(String isbn10, String isbn13, String titre, String auteur)
Récupération des informations sur le livre ComposantBDLivre getLivre(int idLivre)

Liste des abonnés

JSP : WebContent/liste_abonnes.jsp

La page affichant la liste des abonnés est en tous points similaire à celle affichant la liste des livres. Des manipulations sur l'ensemble des données de type recherche, tris, etc. sont possibles.
Pour chacun des abonnés de la liste, un lien (colonne Informations) permet d'accéder à une page autorisant la manipulation de celui-ci : modification, etc.

Une seule méthode est nécessaire :

Emprunter un exemplaire

JSP : WebContent/emprunter_exemplaire.jsp

L'emprunt d'un exemplaire se fait par l'intermédiare d'un simple formulaire dans lequel sont saisis l'identifiant de l'exemplaire emprunté et l'identifiant de l'abonné emprunteur.

Une fois l'emprunt réalisé, la page affiche les informations liées à l'emprunt :

  • Le titre du livre associé à l'exemplaire
  • L'identifiant de l'exemplaire
  • Le nom et le prénom de l'emprunteur

Les méthodes nécessaires sont :

Classe Méthode
Emprunt d'un exemplaire par un abonné ComposantBDEmprunt emprunter(int idAbonne, int idExemplaire)

Récupération des informations sur l'abonné ComposantBDAbonne getAbonne(int idAbonne)

Retourner un exemplaire

JSP : WebContent/retourner_exemplaire.jsp

Le retour d'un exemplaire se fait par l'intermédiare d'un simple formulaire dans lequel est saisi l'identifiant de l'exemplaire à retourner.

Cette page peut être appelée à partir du menu de l'application mais est est aussi appelée à partir d'autres pages de l'application (liste des emprunts). Dans ce cas, la valeur de l'identifiant de l'exemplaire est automatiquement renseignée dans le formulaire : voir le paramètre dans l'URL d'appel de la page.

Une fois le retour réalisé, la page affiche les informations suivantes :

  • Le titre du livre associé à l'exemplaire
  • L'identifiant de l'exemplaire

Les méthodes nécessaires sont :

Classe Méthode
Retour d'un exemplaire ComposantBDEmprunt rendre(int idExemplaire)
Récupération des informations du livre associé à l'exemplaire ComposantBDLivre getLivreParIdExemplaire(int idExemplaire)
Récupération des informations sur l'abonné ComposantBDAbonne getAbonne(int idAbonne)

Statistiques des emprunts

JSP : WebContent/accueil.jsp
JSP : WebContent/fragments/fin_de_page.jsp

Travail optionnel

Les statistiques des emprunts (un bien grand mot...) s'affichent sur la page d'accueil : il s'agit du graphique intitulé Mouvements de la bibliothèque qui représente les deux courbes affichant jour par jour le nombre d'emprunts et le nombre de retours.

Par défaut cette fonctionnalité est désactivée. Pour pouvoir l'afficher, il est nécessaire de décommenter l'affichage du graphique dans le JSP accueil.jsp : suppression aux lignes 86 et 95 des marques de début et de fin de commentaires : <%-- et --%>.
Une fois cette étape réalisée, le cadre Mouvements de la bibliothèque s'affiche mais sans les deux courbes, le code permettant l'affichage de celles-ci ne dispose encore d'aucune donnée à afficher.

Pour récupérer les données nécessaires à l'affichage des courbes, la méthode suivante est nécessaire :

Classe Méthode
Statistique des emprunts ComposantBDEmprunt statsEmprunts()
  • Note 1
    La méthode statsEmprunts() retourne une structure de données complexe de type Map : HashMap<String, int[]>. Ce type de structures de données stockent des couples (clé --> valeur).
    Ici, la clé est une chaîne de caractère (String) : la date, par exemple : "2016-12-07". La valeur associée est un tableau de deux entiers (int[]) stockant le nombre d'emprunts à l'indice 0 et le nombre de retour à l'indice 1.
    Pour plus d'informations sur ce type de structures de données, voir par exemple : https://openclassrooms.com/courses/java-et-les-collections/l-interface-map-k-v.
  • Note 2
    Les données présentes dans la structure Java au retour de l'appel de la méthode statsEmprunts() ont besoin d'être traduites dans un format texte afin d'être manipulées par le graphique. C'est la raison d'être du code présent au début du JSP WebContent/fragments/fin_de_page.jsp.
  • Note 3
    L'affichage du graphique est dévolue à une bibliothèque Javascript (bibliothèque morris.js), l'appel est réalisé à la fin du JSP WebContent/fragments/fin_de_page.jsp.
    Pour plus d'informations sur cette librairie d'affichage de graphiques pour le Web : http://morrisjs.github.io/morris.js/.
